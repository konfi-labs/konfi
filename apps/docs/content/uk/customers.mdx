# Клієнти

Записи клієнтів зберігають контактний, платіжний та операційний контекст для
замовлень і активності магазину.

## Знайти клієнта

- Відкрий **Клієнти** в основній навігації.
- Шукай за назвою клієнта.
- Онови список, якщо його міг змінити інший оператор.
- Перед зміною спільних даних відкрий перегляд або деталі.

## Керування записами

- **Додати клієнта** створює новий запис.
- Дії рядка дозволяють редагувати, дублювати або деактивувати клієнта.
- Перед замовленням чи рахунком перевір назву, контактну особу, платіжні дані
  та ідентифікатори.

## Групи і кошики

- **Групи клієнтів** сегментують клієнтів для операційних правил.
- Клієнта можна прив'язати до групи з дії рядка.
- **Кошики клієнтів** показують контекст з інтернет-магазину.

## Нотатки

- Нотатку можна створити з дій рядка клієнта.
- Вони підходять для інформації про акаунт, яка не має жити лише в одному
  замовленні.

## Безпечні зміни

- Дублюй клієнта для схожого, але окремого акаунта.
- Історичний запис краще деактивувати, ніж перезаписувати його ідентичність.
- Перед зміною платіжних полів перевір пов'язані замовлення.
